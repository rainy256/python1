## ★3 自然语言记账 — 详细实现

### 核心文件
- [validators.py](SmartExpenseTracker/validators.py) — `parse_natural_input()` 函数
- [main.py](SmartExpenseTracker/main.py) — `natural_language_flow()` 函数 + 菜单选项7

### 解析能力

#### 1. 相对日期
| 输入 | 解析结果 |
|------|---------|
| 今天午饭花了35元 | 当天日期 |
| 昨天打车30块 | 昨天日期 |
| 前天理发50 | 前天日期 |

#### 2. 具体日期
| 输入 | 解析结果 |
|------|---------|
| 5月10日买衣服花了299 | 2026-05-10 |

#### 3. 关键词分类（40+关键词 → 9个分类）
| 关键词 | 映射分类 |
|--------|---------|
| 吃饭/午饭/晚饭/外卖/聚餐 | 餐饮 |
| 地铁/公交/打车/加油 | 交通 |
| 买/衣服/淘宝/京东 | 购物 |
| 电影/游戏/KTV/旅游 | 娱乐 |
| 房租/水电/物业 | 居住 |
| 医院/药 | 医疗 |
| 书/课程/培训 | 教育 |
| 工资/奖金/报销 | 工资（收入） |

#### 4. 三层金额兜底匹配
```
第一层：带单位 — 35元 / 30块 / 1万
第二层：关键词后 — 花了299 / 收入8000 / 用了50
第三层：末尾裸数字 — 理发50
```

#### 5. 自动收支分类
- 包含"工资/奖金/报销/收入" → 自动识别为**收入**
- 其余 → **支出**

### 演示示例
```
输入: 昨天午饭花了35元
→ 2026-05-13 餐饮 35.00元 (支出)

输入: 今天工资收入8000
→ 2026-05-14 工资 8000.00元 (收入)

输入: 5月10日买衣服花了299
→ 2026-05-10 购物 299.00元 (支出)

输入: 打车30块
→ 2026-05-14 交通 30.00元 (支出)

输入: 房租交了2000
→ 2026-05-14 居住 2000.00元 (支出)
```

## ★1 异常消费检测 — 详细实现

### 核心文件
- [anomaly.py](SmartExpenseTracker/anomaly.py) — 检测模块
- [main.py](SmartExpenseTracker/main.py) — `anomaly_check_flow()` 函数 + 菜单选项8
- [main.py](SmartExpenseTracker/main.py) — 选项3（查看分类统计）末尾自动触发

### 检测机制（双轨制）

#### 轨道1：均值异常检测
```
算法：本月消费 > 前3个月月均消费 × 2.0倍 → 标记异常
条件：至少有2个月历史数据
```

#### 轨道2：新增大额检测
```
算法：前3个月该分类无任何记录 + 本月消费 ≥ 200元 → 提醒
用途：发现新出现的消费类型
```

### 触发方式（两处）

| 触发点 | 菜单选项 | 方式 |
|--------|:---:|------|
| 手动检测 | 菜单 **8** | 独立报告，有异常显示详情，无异常显示"一切正常" |
| 自动检测 | 菜单 **3** | 查看分类统计后自动连带检测 |

### 演示效果
```
=============================================
  ⚠️  消费异常检测报告
=============================================

📊 基于历史均值的异常消费:
  分类             本月消费        历史均值        超出倍数
  ------------------------------------------
  购物           550.00       50.00       11.0x
    → 近3个月均消费50.0元，本月消费大幅超出！

🆕 新增大额消费提醒:
  娱乐: 本月消费300.00元，前3个月无此分类记录
=============================================
```

### 无异常时
```
🎉 本月消费一切正常，未发现异常！
```
